# NIST SP 800-61 Rev. 2 - Computer Security Incident Handling Guide

## Incident Response Lifecycle

1. **Preparation** - Establish policies, tools, and training
2. **Detection and Analysis** - Identify and analyze indicators
3. **Containment, Eradication, and Recovery** - Limit damage, remove threats, restore
4. **Post-Incident Activity** - Lessons learned, process improvement

## Key Principles

- Prioritize based on business impact
- Preserve evidence for legal proceedings
- Document all actions with timestamps
- Coordinate communication with stakeholders

## Incident Classification

| Category | Description | Examples |
|----------|-------------|----------|
| Denial of Service | Attacks that prevent resource availability | DDoS, resource exhaustion |
| Malicious Code | Viruses, worms, trojans | Ransomware, backdoors |
| Unauthorized Access | Logical or physical access without permission | Hacking, credential theft |
| Inappropriate Usage | Violation of acceptable use policies | Policy violations |
